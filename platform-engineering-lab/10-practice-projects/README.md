# 实战项目：平台工程综合训练

> 理论知识需要通过实战来巩固。本章提供 6 个可执行的实战项目，
> 覆盖从 IDP 原型到生产排障的完整平台工程技能栈。
> 每个项目都包含完整的部署脚本、故障注入工具、诊断脚本和评分标准。
> 完成所有项目相当于具备 P7-P8 级别平台工程师的实战能力。

## 常见问题

```
Q: 硬件不满足要求怎么办？
A:
  - 可以使用云厂商的免费额度（AWS/Azure/GCP 都有 $200+ 免费额度）
  - 或减少集群规模（单节点 Kind 集群）
  - 部分项目可以按需执行，不需要同时运行所有组件
  - 项目 2 可以从 2 集群减为 1 集群（只对比配置差异）

Q: 遇到错误怎么办？
A:
  1. 检查日志：kubectl logs <pod>
  2. 检查事件：kubectl describe <resource>
  3. 检查资源：kubectl top nodes/pods
  4. 检查节点：kubectl get nodes -o wide
  5. 参考排障手册（11-production-troubleshooting）
  6. 在 GitHub Issue 区提问

Q: 完成所有项目需要多久？
A:
  - 有 K8s 基础：40-60 小时
  - 无 K8s 基础：80-120 小时
  - 建议每天 2-3 小时，分 3-4 周完成
  - 项目 2、3、4 可以并行学习（独立环境）

Q: 如何验证自己的水平？
A:
  - 能够独立完成项目 2-5 中的任意 2 个：P6-P7 水平
  - 能够独立完成所有项目 + 提出优化方案：P7-P8 水平
  - 能够修改源码或设计新架构：P8+ 水平
```

## 面试常见问题补充

**Q: 这个项目集合和实际工作经验有什么区别？**

A: 区别:
- **项目集合**: 受控环境，已知问题，有标准答案
- **实际工作**: 未知问题，没有标准答案，需要权衡

弥补方法:
- 在完成项目后，尝试在现有工作中应用所学
- 主动寻找"土路"（非平台标准路径），思考如何铺成"路"
- 参与开源社区，解决真实问题

**Q: 如何向面试官展示项目经验？**

A: STAR + 量化:
- **Situation**: "在平台工程实验室项目中，我负责搭建多租户 K8s 平台"
- **Task**: "需要为 3 个团队提供隔离环境，同时保持成本可控"
- **Action**: "使用 vCluster 创建虚拟集群，配合 ResourceQuota 和 NetworkPolicy"
- **Result**: "成功部署，每个团队有独立环境，资源利用率提升 30%，成本降低 20%"

准备材料:
- GitHub 仓库链接（展示代码）
- 架构图（展示设计能力）
- 性能数据（展示量化思维）
- 故障案例（展示排障能力）
