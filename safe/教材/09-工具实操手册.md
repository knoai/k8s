> **本章目标**：掌握 Kubernetes 安全生态中所有核心工具的详细操作方法。我们将从安装配置到高级用法，提供完整的操作指南。

## 9.1 Trivy：全能漏洞扫描器

### 9.1.1 安装与基础配置

```bash
# macOS
brew install trivy

# Linux
curl -sfL https://raw.githubusercontent.com/aquasecurity/trivy/main/contrib/install.sh | sh -s -- -b /usr/local/bin

# Docker
docker pull aquasec/trivy:latest

# 验证安装
trivy version
```

### 9.1.2 镜像扫描

```bash
# 基本扫描
trivy image nginx:alpine

# 只显示 HIGH 和 CRITICAL
trivy image --severity HIGH,CRITICAL nginx:alpine

# 忽略未修复的漏洞
trivy image --ignore-unfixed nginx:alpine

# 扫描特定操作系统包和语言包
trivy image --vuln-types os,library nginx:alpine

# 输出为 SARIF（GitHub/GitLab Security Dashboard）
trivy image -f sarif -o report.sarif nginx:alpine

# 输出为 JSON
trivy image -f json -o report.json nginx:alpine

# 输出为表格（默认）
trivy image -f table nginx:alpine

# 跳过更新漏洞数据库（离线环境）
trivy image --skip-db-update nginx:alpine

# 使用离线漏洞数据库
trivy image --offline-scan nginx:alpine

# 扫描结果解释：
# | Library | Vulnerability | Severity | Installed | Fixed Version | Title |
# |---------|--------------|----------|-----------|---------------|-------|
# | openssl | CVE-2023-xxx | HIGH     | 1.1.1w    | 1.1.1x        | ...   |
```

### 9.1.3 文件系统扫描

```bash
# 扫描项目文件系统
trivy filesystem /path/to/project

# 扫描特定类型漏洞
trivy filesystem --scanners vuln,secret,config /path/to/project

# 扫描结果示例（Secret 检测）：
# | Type    | Rule ID     | Severity | Category  | Match                    |
# |---------|-------------|----------|-----------|--------------------------|
# | AWS     | aws-access-key-id | HIGH | API Key   | AKIAIOSFODNN7EXAMPLE    |
```

### 9.1.4 Kubernetes 扫描

```bash
# 扫描整个集群（资源漏洞）
trivy k8s cluster --report summary

# 扫描特定命名空间
trivy k8s --namespace production --report all

# 扫描特定资源
trivy k8s deployment/nginx -n production

# 输出为 JSON
trivy k8s cluster --report all -f json -o k8s-scan.json
```

### 9.1.5 IaC 扫描

```bash
# 扫描 Kubernetes manifest
trivy config /path/to/k8s-manifests/

# 扫描 Terraform
trivy config /path/to/terraform/

# 扫描 Dockerfile
trivy config /path/to/Dockerfile

# 扫描结果示例：
# | Misconfiguration ID | Severity | Message | Resolution |
# |---------------------|----------|---------|------------|
# | KSV001 | HIGH | Privileged container | Set privileged=false |
```

### 9.1.6 SBOM 生成与扫描

```bash
# 生成 SPDX SBOM
trivy image --format spdx-json -o sbom.spdx.json nginx:alpine

# 生成 CycloneDX SBOM
trivy image --format cyclonedx-json -o sbom.cdx.json nginx:alpine

# 从 SBOM 扫描漏洞
trivy sbom sbom.spdx.json

# 生成并附加到镜像（支持 OCI 仓库）
trivy image --format spdx-json -o sbom.json nginx:alpine
cosign attach sbom --sbom sbom.json myregistry/nginx:alpine
```

### 9.1.7 CI/CD 集成

```yaml
# GitHub Actions 示例
- name: Run Trivy vulnerability scanner
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: 'myapp:${{ github.sha }}'
    format: 'sarif'

    severity: 'CRITICAL,HIGH'
    exit-code: '1'    # 发现漏洞则失败

- name: Upload Trivy scan results
  uses: github/codeql-action/upload-sarif@v2
  with:
    sarif_file: 'trivy-results.sarif'
```

## 9.2 Falco：运行时威胁检测

### 9.2.1 安装与配置

```bash
# Helm 安装（Modern eBPF 驱动）
helm repo add falcosecurity https://falcosecurity.github.io/charts
helm install falco falcosecurity/falco \
  -n falco --create-namespace \
  --set driver.kind=modern_ebpf \
  --set collectors.enabled=true \
  --set tty=true

# 查看日志
kubectl logs -n falco -l app.kubernetes.io/name=falco -f

# 查看 Falco 版本和规则列表
kubectl exec -n falco -it deployment/falco -- falco --version
kubectl exec -n falco -it deployment/falco -- falco --list
```

### 9.2.2 规则编写指南

```yaml
# Falco 规则结构
- rule: <规则名称>
  desc: <描述>
  condition: <过滤条件>
  output: <输出格式>
  priority: <优先级>
  enabled: <是否启用>
  tags: [<标签列表>]

# 常用条件字段：
# 进程：proc.name, proc.cmdline, proc.pid, proc.ppid
# 容器：container.name, container.image, container.privileged
# 文件：fd.name, fd.directory
# 网络：fd.sip, fd.sport
# K8s：k8s.ns.name, k8s.pod.name, ka.user.name
# 用户：user.name, user.uid
```

### 9.2.3 自定义规则示例

```yaml
# custom-rules.yaml
# 检测反向 shell
- rule: Reverse shell
  desc: Detect reverse shell connections
  condition: >
    spawned_process
    and (shell_procs or exec_bin)
    and (fd.sip.name != 127.0.0.1 and fd.sip.name != ::1)
  output: >
    Reverse shell connection detected
    (command=%proc.cmdline connection=%fd.name user=%user.name)
  priority: CRITICAL
  tags: [network, shell, mitre_execution]

# 检测敏感文件修改
- rule: Write below etc
  desc: Detect write operations under /etc
  condition: >
    write_etc_common
    and not proc.name in ("iptables", "ip6tables", "systemd")
  output: >
    File written to /etc
    (file=%fd.name command=%proc.cmdline user=%user.name)
  priority: WARNING

# 宏定义
- macro: spawned_process
  condition: (evt.type in (execve, execveat) and evt.dir=<)

# 列表定义
- list: shell_procs
  items: [bash, sh, zsh, ash, csh, tcsh, ksh, dash]
```

### 9.2.4 输出配置

```yaml
# /etc/falco/falco.yaml
json_output: true
file_output:
  enabled: true
  keep_alive: false
```

## 9.3 Kyverno：K8s 原生策略引擎

### 9.3.1 安装

```bash
helm repo add kyverno https://kyverno.github.io/kyverno/
helm install kyverno kyverno/kyverno \
  -n kyverno --create-namespace \
  --set admissionController.replicas=3
```

### 9.3.2 策略类型

| 类型 | 说明 | 用途 |
|------|------|------|
| **Validate** | 验证资源是否符合规则 | 拒绝不合规资源 |
| **Mutate** | 自动修改资源 | 添加标签、注入 sidecar |
| **Generate** | 自动生成关联资源 | 创建 NetworkPolicy、Secret |
| **VerifyImages** | 验证镜像签名 | Cosign 签名验证 |

### 9.3.3 策略示例库

```yaml
# 1. 禁止特权容器
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: disallow-privileged
spec:
  validationFailureAction: Enforce
  rules:
  - name: check-privileged
    match:
      resources:
        kinds:
        - Pod
    validate:
      message: "Privileged containers are not allowed"
      pattern:
        spec:
          containers:
          - securityContext:
              =(privileged): "false"

# 2. 强制使用特定仓库
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: restrict-image-registries
spec:
  validationFailureAction: Enforce
  rules:
  - name: validate-registries
    match:
      resources:
        kinds:
        - Pod
    validate:
      message: "Only approved registries allowed"
      pattern:
        spec:
          containers:
          - image: "myregistry.io/* | gcr.io/company/* | ghcr.io/company/*"

# 3. 自动添加资源限制
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: add-resource-limits
spec:
  rules:
  - name: add-limits
    match:
      resources:
        kinds:
        - Pod
    mutate:
      patchStrategicMerge:
        spec:
          containers:
          - (name): "*"
            resources:
              limits:
                memory: "256Mi"
                cpu: "500m"
              requests:
                memory: "128Mi"
                cpu: "100m"

# 4. 自动添加 NetworkPolicy
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: generate-default-networkpolicy
spec:
  rules:
  - name: default-deny
    match:
      resources:
        kinds:
        - Namespace
    generate:
      kind: NetworkPolicy
      apiVersion: networking.k8s.io/v1
      name: default-deny
      namespace: "{{request.object.metadata.name}}"
      data:
        spec:
          podSelector: {}
          policyTypes:
          - Ingress
          - Egress
```

### 9.3.4 CLI 工具

```bash
# 测试策略（不应用到集群）
kyverno test /path/to/test-dir

# 扫描现有资源
kyverno apply /path/to/policies --resource /path/to/resource.yaml

# 验证策略语法
kyverno validate /path/to/policy.yaml
```

## 9.4 OPA Gatekeeper

### 9.4.1 安装

```bash
helm repo add gatekeeper https://open-policy-agent.github.io/gatekeeper/charts
helm install gatekeeper gatekeeper/gatekeeper \
  -n gatekeeper-system --create-namespace
```

### 9.4.2 Rego 策略示例

```rego
# 禁止 latest 标签
package k8srequiredtags
violation[{"msg": msg}] {
  container := input.review.object.spec.containers[_]
  endswith(container.image, ":latest")
  msg := sprintf("Container %v uses latest tag", [container.name])
}

# 禁止 hostPath
package k8sprohibitedvolume
violation[{"msg": msg}] {
  volume := input.review.object.spec.volumes[_]
  volume.hostPath
  msg := sprintf("Volume %v uses hostPath", [volume.name])
}

# 要求资源限制
package k8srequiredresources
violation[{"msg": msg}] {
  container := input.review.object.spec.containers[_]
  not container.resources.limits.memory
  msg := sprintf("Container %v missing memory limit", [container.name])
}
```

### 9.4.3 约束模板与约束

```yaml
# 约束模板
apiVersion: templates.gatekeeper.sh/v1
kind: ConstraintTemplate
metadata:
  name: k8srequiredlabels
spec:
  crd:
    spec:
      names:
        kind: K8sRequiredLabels
      validation:
        openAPIV3Schema:
          type: object
          properties:
            labels:
              type: array
              items:
                type: string
  targets:
  - target: admission.k8s.gatekeeper.sh
    rego: |
      package k8srequiredlabels
      violation[{"msg": msg}] {
        provided := {label | input.review.object.metadata.labels[label]}
        required := {label | label := input.parameters.labels[_]}
        missing := required - provided
        count(missing) > 0
        msg := sprintf("Missing required labels: %v", [missing])
      }

# 约束
---
apiVersion: constraints.gatekeeper.sh/v1beta1
kind: K8sRequiredLabels
metadata:
  name: require-team-label
spec:
  match:
    kinds:
    - apiGroups: [""]
      kinds: ["Pod", "Deployment"]
    namespaces: ["production"]
  parameters:
    labels:
    - team
    - environment
```

## 9.9 Tetragon：eBPF 运行时安全

### 9.9.1 安装

```bash
# Helm 安装
helm repo add cilium https://helm.cilium.io/
helm install tetragon cilium/tetragon -n kube-system

# 验证
kubectl rollout status -n kube-system ds/tetragon
kubectl logs -n kube-system -l app.kubernetes.io/name=tetragon -c export-stdout -f
```

### 9.9.2 核心概念

| 概念 | 说明 | 对比 Falco |
|------|------|-----------|
| **TracingPolicy** | 定义跟踪策略（YAML） | 类似 Falco 规则 |
| **eBPF 内核探针** | kprobe/tracepoint 级跟踪 | Falco 使用 tracepoints |
| **自动响应** | 可配置自动终止进程 | Falco 只告警不响应 |
| **无 Sidecar** |  DaemonSet 部署，无 per-Pod 开销 | Falco 也是 DaemonSet |

### 9.9.3 TracingPolicy 示例

```yaml
# 检测并终止特权容器内的敏感操作
apiVersion: cilium.io/v1alpha1
kind: TracingPolicy
metadata:
  name: "monitor-sensitive-files"
spec:
  kprobes:
  - call: "sys_openat"
    syscall: true
    args:
    - index: 0
      type: "int"
    - index: 1
      type: "string"
    - index: 2
      type: "int"
    selectors:
    - matchBinaries:
      - operator: "In"
        values:
        - "/bin/sh"
        - "/bin/bash"
      matchArgs:
      - index: 1
        operator: "Prefix"
        values:
        - "/etc/shadow"
        - "/etc/kubernetes/admin.conf"
      matchActions:
      - action: Sigkill    # 自动终止进程！
        message: "Sensitive file access detected and blocked"

---
# 检测容器逃逸尝试
apiVersion: cilium.io/v1alpha1
kind: TracingPolicy
metadata:
  name: "detect-mount-escape"
spec:
  kprobes:
  - call: "sys_mount"
    syscall: true
    args:
    - index: 0
      type: "string"
    - index: 1
      type: "string"
    - index: 2
      type: "string"
    selectors:
    - matchArgs:
      - index: 1
        operator: "Prefix"
        values:
        - "/host"
        - "/proc/1/root"
      matchActions:
      - action: Sigkill
        message: "Container escape attempt blocked"
```

### 9.9.4 与 Falco 的对比

| 特性 | Falco | Tetragon |
|------|-------|----------|
| **检测粒度** | 系统调用 + 审计事件 | kprobe 级内核函数 |

## 9.13 安全加固检查清单脚本

```bash
#!/bin/bash
# k8s-security-audit.sh - 自动化安全检查脚本

set -e

echo "=== K8s 安全检查脚本 ==="
echo "运行时间: $(date)"
echo ""

PASS=0
FAIL=0
WARN=0

check() {
  local name="$1"
  local cmd="$2"
  local expected="$3"
  
  echo -n "[检查] $name ... "
  if eval "$cmd" >/dev/null 2>&1; then
    if [ "$expected" = "true" ]; then
      echo "✅ PASS"
      ((PASS++))
    else
      echo "❌ FAIL (预期不存在但存在)"
      ((FAIL++))
    fi
  else
    if [ "$expected" = "false" ]; then
      echo "✅ PASS"
      ((PASS++))
    else
      echo "❌ FAIL (预期存在但不存在)"
      ((FAIL++))
    fi
  fi
}

# 1. 匿名认证
check "API Server 匿名认证已禁用" \
  "kubectl get pod -n kube-system -l component=kube-apiserver -o yaml | grep -q 'anonymous-auth=false'" \
  "true"

# 2. etcd 加密
echo -n "[检查] etcd 静态加密已启用 ... "
if kubectl get apiserver -o yaml 2>/dev/null | grep -q encryption; then
  echo "✅ PASS"
  ((PASS++))
else
  echo "⚠️ WARN (请手动确认)"
  ((WARN++))
fi

# 3. Pod Security Standards
echo -n "[检查] 命名空间已启用 PSS ... "
PSS_COUNT=$(kubectl get ns -o json | jq '[.items[].metadata.labels | select(."pod-security.kubernetes.io/enforce")] | length')
if [ "$PSS_COUNT" -gt 0 ]; then
  echo "✅ PASS ($PSS_COUNT 个命名空间)"
  ((PASS++))
else
  echo "❌ FAIL (无命名空间启用 PSS)"
  ((FAIL++))
fi

# 4. 特权容器
PRIV_COUNT=$(kubectl get pods -A -o json 2>/dev/null | \
  jq '[.items[].spec.containers[]?.securityContext?.privileged] | map(select(.==true)) | length' || echo "0")
if [ "$PRIV_COUNT" = "0" ]; then
  echo "✅ [检查] 特权容器: 0 个"
  ((PASS++))
else
  echo "⚠️ [检查] 特权容器: $PRIV_COUNT 个"
  ((WARN++))
fi

# 5. ServiceAccount 自动挂载
echo -n "[检查] 默认 ServiceAccount 已禁用自动挂载 ... "
SA_COUNT=$(kubectl get sa -A -o json | \
  jq '[.items[] | select(.metadata.name=="default" and .automountServiceAccountToken!=false)] | length')
if [ "$SA_COUNT" = "0" ]; then
  echo "✅ PASS"
  ((PASS++))
else
  echo "⚠️ WARN ($SA_COUNT 个默认 SA 仍启用自动挂载)"
  ((WARN++))
fi

# 6. NetworkPolicy
NP_COUNT=$(kubectl get networkpolicy -A --no-headers 2>/dev/null | wc -l | tr -d ' ')
if [ "$NP_COUNT" -gt 0 ]; then
  echo "✅ [检查] NetworkPolicy: $NP_COUNT 条"
  ((PASS++))
else
  echo "⚠️ [检查] NetworkPolicy: 0 条"
  ((WARN++))
fi

# 7. RBAC
echo -n "[检查] cluster-admin 绑定审查 ... "
ADMIN_BINDINGS=$(kubectl get clusterrolebinding -o json 2>/dev/null | \
  jq -r '.items[] | select(.roleRef.name=="cluster-admin") | .metadata.name' | wc -l | tr -d ' ')
echo "发现 $ADMIN_BINDINGS 个 cluster-admin 绑定"

# 8. 审计日志
check "API Server 审计日志已启用" \
  "kubectl get pod -n kube-system -l component=kube-apiserver -o yaml | grep -q 'audit-log-path'" \
  "true"

# 总结
echo ""
echo "=== 检查完成 ==="
echo "✅ PASS: $PASS"
echo "❌ FAIL: $FAIL"
echo "⚠️ WARN: $WARN"

if [ "$FAIL" -gt 0 ]; then
  echo ""
  echo "建议优先修复 FAIL 项"
  exit 1
fi
```

## 9.13 常见问题与故障排查

### 9.13.1 Trivy 扫描问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `TOOMANYREQUESTS` | Docker Hub 速率限制 | 配置镜像仓库认证或镜像缓存 |
| 扫描结果为空 | 使用的数据库过期 | `trivy image --download-db-only` |
| 内存不足 | 大型镜像扫描 | 增加内存或使用 `--skip-dirs` |
| 扫描速度慢 | 未使用缓存 | 启用 `--cache-dir` 持久化缓存 |

### 9.13.2 Falco 运行问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Falco 无法启动 | 驱动加载失败 | 检查内核版本，使用 modern_ebpf |
| 无告警输出 | 规则不匹配 | 检查规则语法，使用 `falco -r` 验证 |
| 大量误报 | 规则过于宽泛 | 细化规则条件，添加排除项 |
| 性能问题 | 规则过多 | 精简规则集，使用预过滤 |

### 9.13.3 Kyverno 策略问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 策略不生效 | webhook 未注册 | 检查 kyverno Pod 状态，查看证书 |
| 拒绝所有请求 | 策略过于严格 | 设置 `validationFailureAction: Audit` 先观察 |
| 变量不解析 | JMESPath 语法错误 | 使用 `kyverno jp query` 测试 |
| 背景扫描失败 | 权限不足 | 检查 kyverno ServiceAccount RBAC |

```yaml
        output: 'trivy-results.sarif'
        severity: 'CRITICAL,HIGH'
    
    - name: Upload SARIF results
      uses: github/codeql-action/upload-sarif@v2
      with:
        sarif_file: 'trivy-results.sarif'

  k8s-config-scan:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Scan K8s manifests
      uses: aquasecurity/trivy-action@master
      with:
        scan-type: 'config'
        scan-ref: './k8s/'
        format: 'table'
        exit-code: '1'
        severity: 'CRITICAL,HIGH'
```

### 9.15.1 每日安全巡检命令速查

```bash
# === 30 秒快速巡检 ===

# 1. 异常 Pod（非 Running）
kubectl get pods -A | grep -v Running | grep -v Completed

# 2. 特权容器
kubectl get pods -A -o json | jq -r '.items[] | select(.spec.containers[]?.securityContext?.privileged==true) | "\(.metadata.namespace)/\(.metadata.name)"'

# 3. 高危 RBAC
kubectl get clusterrolebinding -o json | jq -r '.items[] | select(.roleRef.name=="cluster-admin") | "\(.metadata.name): \(.subjects | map(.name) | join(", "))"'

# 4. 无 NetworkPolicy 命名空间
for ns in $(kubectl get ns -o jsonpath='{.items[*].metadata.name}'); do
  [ "$ns" != "kube-system" ] && [ "$(kubectl get netpol -n "$ns" --no-headers 2>/dev/null | wc -l)" -eq 0 ] && echo "$ns"
done

# 5. 证书过期（30天内）
kubeadm certs check-expiration | grep -E "(EXPIRES|less than)"

# 6. Falco 最近告警
kubectl logs -n falco -l app.kubernetes.io/name=falco --tail=50 | grep "Warning\|Error\|Critical"

# 7. 高危镜像（Top 5）
trivy image --severity HIGH,CRITICAL --quiet <IMAGE> 2>/dev/null | grep -E "^(High|Critical)" | head -5
```

## 9.15 本章小结

| 工具 | 主要功能 | 关键命令 | 适用场景 |
|------|---------|---------|---------|
| **Trivy** | 漏洞扫描 | `trivy image/fs/k8s/config/sbom` | 镜像、代码、K8s、IaC |

| **kube-bench** | CIS 合规 | `run --check` | 节点合规检查 |
