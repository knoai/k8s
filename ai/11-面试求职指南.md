## 三、高频面试题（按类别）

### 3.1 K8s 基础（15 题）

1. **Pod 生命周期有哪些状态？什么情况下会进入 Pending？**
   > Pending → ContainerCreating → Running → Succeeded/Failed/CrashLoopBackOff。Pending 原因：资源不足、镜像拉取中、调度失败、PVC 未绑定。

2. **Deployment 和 StatefulSet 的区别？**
   > Deployment：无状态，随机后缀名，共享存储。StatefulSet：有序部署，稳定网络标识（pod-name-N），独立 PVC。

3. **K8s 调度器的工作原理？**
   > Watch 未调度 Pod → Predicates 过滤（资源/亲和性/污点）→ Priorities 评分 → 选择最优节点 → 绑定。

4. **Resource Request 和 Limit 的区别？**
   > Request 用于调度决策，Limit 是运行时的硬限制。CPU 超 Limit 会被节流，内存超 Limit 会被 OOMKill。

5. **Init Container 和 Sidecar 的使用场景？**
   > Init：主容器启动前初始化（如等待依赖、配置生成）。Sidecar：与主容器并行运行（如日志收集、代理、监控）。

6. **Service 的 ClusterIP 原理？**
   > kube-proxy 维护 iptables/IPVS 规则，将 ClusterIP 映射到后端 Pod IP。DNS 通过 CoreDNS 解析。

7. **Ingress 和 Gateway API 的区别？**
   > Ingress 是 L7 路由的最初实现，功能有限。Gateway API 是下一代标准，支持 TCP/UDP/HTTP，更灵活，由 Envoy/Cilium 等实现。

8. **ConfigMap 和 Secret 的区别？如何安全使用 Secret？**
   > ConfigMap 存明文配置，Secret 存敏感数据（Base64 编码，非加密）。安全：启用 etcd 加密、RBAC 限制访问、使用外部 Secret 管理（Vault）。

9. **RBAC 中 Role 和 ClusterRole 的区别？**
   > Role 作用于命名空间，ClusterRole 作用于整个集群。RoleBinding 绑定 Role，ClusterRoleBinding 绑定 ClusterRole。

10. **Helm Chart 的 values.yaml 如何覆盖？**
    > `helm install -f custom-values.yaml` 或 `--set key=value`。

11. **K8s 中如何实现灰度发布？**
    > Deployment 的 RollingUpdate + maxSurge/maxUnavailable。高级：Argo Rollouts、Flagger、KServe Canary。

12. **etcd 备份和恢复怎么做？**
    > 备份：`etcdctl snapshot save backup.db`。恢复：停止 etcd → `etcdctl snapshot restore` → 重启。

13. **K8s 中如何实现多租户隔离？**
    > Namespace + RBAC + NetworkPolicy + ResourceQuota + LimitRange + HAMi/MIG。

14. **什么是 Operator 模式？**
    > CRD + Controller。自定义资源描述期望状态，Controller 通过 Reconcile 循环使实际状态趋近期望状态。

15. **kubectl apply 和 kubectl create 的区别？**
    > create 只能创建新资源，apply 使用声明式配置，可以创建或更新，支持增量更新。

### 3.2 GPU 与调度（10 题）

16. **K8s 中 GPU 调度是如何实现的？Device Plugin 机制是什么？**
    > Device Plugin 以 DaemonSet 运行，通过 gRPC 向 kubelet 注册设备。kubelet 上报 API Server，调度器分配，`nvidia.com/gpu` 是扩展资源。

17. **Gang Scheduling 是什么？为什么在 AI 训练中很重要？**
    > 全有或全无调度，要么所有 Worker 都启动，要么都不启动。避免分布式训练中部分 Worker 启动、部分 Pending 导致的资源死锁。

18. **Volcano 和默认 K8s 调度器的区别？**
    > Volcano 支持 Gang Scheduling、Queue、优先级抢占、拓扑感知。默认调度器不支持这些。

19. **如何在 K8s 上实现 GPU 共享？vGPU / MIG / Time Slicing 对比？**
    > vGPU（HAMi）：显存隔离，任意比例切分。MIG：硬件级隔离，固定配置。Time Slicing：进程级时间片，无隔离。

20. **拓扑感知调度是什么？如何优化多节点训练的网络性能？**
    > 将 Pod 调度到网络拓扑相近的节点（同交换机/同机架）。优化：NVLink（单机）、InfiniBand（多机）、NCCL 调优、拓扑感知调度。

21. **Kueue 的 ClusterQueue 和 LocalQueue 是什么关系？**
    > ClusterQueue 定义集群级资源池，LocalQueue 是命名空间级入口，Workload 提交到 LocalQueue，最终由 ClusterQueue 分配资源。

22. **GPU 节点出现 Xid Error 怎么处理？**
    > Xid 是 NVIDIA GPU 错误码。常见：Xid 48（ECC 错误）、Xid 74（NVLink 错误）。处理：查看 DCGM 日志，重启 Pod，严重时节点下线维修。

23. **MIG 和 HAMi 的选择？**
    > A100/H100 生产环境用 MIG（硬隔离）。通用场景/多型号 GPU 用 HAMi（灵活）。

24. **如何防止 GPU 训练任务被推理任务抢占？**
    > 优先级类：训练用 `training-batch`（可被抢占），推理用 `inference-critical`（不可被抢占）。或不同队列不同权重。

25. **K8s 中 GPU 的 requests 和 limits 必须相等吗？为什么？**
    > 是的。GPU 是不可分割的物理资源，不支持超售。分数 GPU 需要通过 vGPU/HAMi 实现。

### 3.3 MLOps / Kubeflow（10 题）

26. **Kubeflow Pipelines 的核心概念？**
    > Pipeline：工作流定义。Component：可复用步骤。Artifact：中间产物。Experiment：实验分组。Run：一次执行。

27. **KServe 的 InferenceService 架构？**
    > Predictor（推理核心）+ Transformer（预处理/后处理）+ Explainer（可解释性）。支持 Canary、A/B 测试、自动扩缩。

28. **模型版本管理怎么做？MLflow 的 Model Registry 机制？**
    > MLflow Tracking 记录实验 → Model Registry 注册版本 → Staging → Production → Archived。支持版本切换、A/B 测试。

29. **Katib 支持哪些调参算法？贝叶斯优化原理？**
    > Random、Grid、Bayesian Optimization、Hyperband、TPE、CMA-ES、NAS。贝叶斯：用高斯过程拟合目标函数，采集函数选择下一采样点。

30. **Kubeflow Training Operator 如何管理分布式训练任务？**
    > 通过 CRD（TFJob/PyTorchJob/MPIJob）定义训练任务，Controller 创建对应的 Pod，Gang Scheduling 保证资源原子分配。

31. **MLflow 和 Kubeflow Pipelines 的区别？**
    > MLflow 侧重实验追踪和模型管理。KFP 侧重工作流编排。可以集成：KFP 中调用 MLflow 记录指标。

32. **如何实现模型的 A/B 测试？**
    > KServe Canary/Inference Graph。或 Envoy 权重路由。或应用层根据用户特征分流。

33. **Kubeflow Pipeline 中如何实现条件分支？**
    > `dsl.Condition()` 根据上游输出决定执行路径。

34. **Feature Store 是什么？Feast 的核心概念？**
    > 统一管理特征的定义、存储、服务。Feast：FeatureView（特征视图）、Entity（实体）、FeatureService（服务接口）。

35. **MLOps 和 DevOps 的核心区别？**
    > MLOps = DevOps + 数据版本管理 + 模型版本管理 + 实验追踪 + 数据漂移监控 + 模型退化检测。

### 3.4 LLM 推理（10 题）

36. **vLLM 的 PagedAttention 原理？**
    > 将 KV Cache 分成固定大小的块（Block），非连续存储，按需分配，支持共享（如 system prompt）。类似 OS 虚拟内存管理。

37. **如何优化 LLM 推理的延迟和吞吐？**
    > Continuous Batching、Prefix Caching、KV Cache 量化、投机解码、量化（AWQ/GPTQ）、模型并行（TP/PP）。

38. **量化（INT8/INT4/AWQ/GPTQ）对推理的影响？**
    > INT8：2× 加速，<2% 精度损失。INT4：4× 加速，3-5% 损失。AWQ：保护重要权重，精度更好。GPTQ：逐层量化，压缩率高。

39. **KServe + vLLM 如何实现 Canary 发布？**
    > `canaryTrafficPercent` 控制流量比例。或 Inference Graph 路由。

40. **推理服务的成本优化策略？**
    > 量化模型、Continuous Batching 提高利用率、自动扩缩（KEDA）、Spot 实例、多模型混部（vGPU）、缓存命中。

41. **TTFT 和 TPOT 是什么？如何优化？**
    > TTFT（Time To First Token）：首 Token 延迟。TPOT（Time Per Output Token）：每输出 Token 时间。优化：Prefix Caching 降低 TTFT，Continuous Batching 降低 TPOT。

42. **vLLM 的 Prefix Caching 原理？**
    > 缓存相同前缀（如 system prompt）的 KV Cache，后续相同前缀的请求直接复用，减少计算。

43. **什么是投机解码（Speculative Decoding）？**
    > 用小模型（draft model）快速生成候选 Token，大模型并行验证。接受则加速，拒绝则回退。

44. **TensorRT-LLM 和 vLLM 的对比？**
    > TRT-LLM：NVIDIA 专用，编译时间长，性能极致（尤其 FP8）。vLLM：通用，启动快，生态好，大多数场景首选。

45. **多机推理时 NCCL 通信慢怎么排查？**
    > 检查 IB 网络状态、`NCCL_DEBUG=INFO` 看日志、确认 RDMA 启用、检查拓扑感知调度、测试 `all_reduce_perf`。

### 3.5 分布式训练（5 题）

46. **数据并行 vs 模型并行 vs 流水线并行？**
    > DP：每个 GPU 有完整模型，数据分片。MP：模型层切分到不同 GPU。PP：层间切分，流水线执行。

47. **PyTorch DDP 的 AllReduce 机制？**
    > Ring AllReduce：每个 GPU 只与邻居通信，经过 2*(N-1) 轮完成全局梯度同步。

48. **NCCL 是什么？如何调试 NCCL 问题？**
    > NVIDIA 集合通信库。调试：`NCCL_DEBUG=INFO`，查看日志中的错误信息，检查 IB 网络，测试 `all_reduce_perf`。

49. **DeepSpeed ZeRO Stage 1/2/3 的区别？**
    > Stage 1：优化器状态分片。Stage 2：+ 梯度分片。Stage 3：+ 参数分片，显存节省最多。

50. **Checkpoint 策略如何设计？**
    > 频率：每 N 步或每 X 分钟。异步保存不阻塞训练。保存到分布式存储（S3/JuiceFS）。故障时自动恢复。

### 3.6 系统设计与架构（5 题）

51. **设计一个支持 1000+ GPU 的 AI 训练平台。**
    > 要点：多集群联邦、Volcano 调度、Kueue 队列、JuiceFS 存储、IB 网络、Prometheus 监控、故障自愈、多租户隔离。

52. **如何实现多租户的 GPU 资源隔离与配额管理？**
    > Namespace + ResourceQuota + HAMi vGPU + Kueue ClusterQueue + NetworkPolicy + Kyverno。

53. **模型从开发到上线的完整 MLOps 流程设计。**
    > 数据准备 → 实验开发（Notebook）→ Pipeline 编排 → 自动调参 → 训练 → 评估 → 注册 → 部署（Canary）→ 监控 → 回滚/重训练。

54. **如何监控和告警大规模推理服务？**
    > 指标：QPS、延迟（TTFT/TPOT）、错误率、GPU 利用率、显存、队列深度。告警：P99 延迟突增、错误率>1%、GPU Xid 错误。

55. **Spot 实例用于训练任务的架构设计？**
    > Checkpoint 频繁保存到 S3。Spot 中断信号捕获（AWS 2 分钟预警）。自动重启新实例恢复训练。优先使用可中断的低优先级队列。

```
- 股票：4 年 20 万
- 签字费：5 万
- 总计：~90-100 万
```

### 5.3 谈薪技巧

1. **先了解市场**：脉脉、OfferShow、Levels.fyi
2. **用数据说话**："优化后集群利用率提升 50%，年节省 200 万"
3. **多 offer 比较**：至少拿 2-3 个再决策
4. **总包思维**：不要只看月薪，看全年总包
5. **长期收益**：股票、期权、晋升空间
