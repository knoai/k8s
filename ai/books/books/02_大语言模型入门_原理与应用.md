## 第1章 大语言模型全景图

### 1.1 从NLP到LLM：发展历程

自然语言处理（NLP）经历了几个关键范式转变，每一次转变都深刻地影响了Agent的开发方式。

**范式演进时间线：**

```
2013 ──► Word2Vec / GloVe
         └── 词嵌入：将单词表示为稠密向量
         └── 局限：一词多义无法区分

2017 ──► Transformer (Attention Is All You Need)
         └── 自注意力机制：序列中任意位置可直接交互
         └── 革命性：并行训练、长距离依赖

2018 ──► BERT / GPT-1
         └── 预训练 + 微调范式确立
         └── BERT：双向编码器，理解任务
         └── GPT：单向解码器，生成任务

2019 ──► GPT-2 (1.5B参数)
         └── 零样本能力初显
         └── 无需微调即可执行简单任务

2020 ──► GPT-3 (175B参数)
         └── In-Context Learning涌现
         └── 通过Prompt中的示例学习新任务
         └── 关键能力：Agent的基础

2022 ──► ChatGPT / InstructGPT
         └── RLHF（人类反馈强化学习）
         └── 对齐：让模型遵循指令、有帮助、安全
         └── 对话能力质变

2023 ──► GPT-4 / Claude-2 / Llama-2
         └── 多模态（GPT-4V）
         └── 长上下文（Claude 100K → 200K）
         └── 开源模型可商用（Llama-2）

2024 ──► GPT-4o / Claude 3.5 / Llama 3.1
         └── 原生多模态（GPT-4o）
         └── 超长上下文（Gemini 1M+）
         └── 开源追上闭源（Llama 3.1 405B）
```

### 1.3 Agent开发中的LLM选型策略

```python
from dataclasses import dataclass
from enum import Enum

class TaskComplexity(Enum):
    TRIVIAL = 1      # 问候、简单确认
    SIMPLE = 2       # FAQ、信息提取
    MODERATE = 3     # 多条件查询、简单分析
    COMPLEX = 4      # 多步推理、代码生成
    EXPERT = 5       # 复杂规划、创新设计

@dataclass
class ModelCapability:
    name: str
    context_window: int
    input_cost_per_1k: float   # 美元
    output_cost_per_1k: float
    reasoning_score: int       # 1-10
    tool_reliability: float    # 0-1
    speed_score: int           # 1-10

class ModelRouter:
    """
    智能模型路由器：根据任务复杂度、预算和质量要求动态选择模型。
    """
    
    MODELS = {
        "gpt-4o-mini": ModelCapability(
            name="gpt-4o-mini", context_window=128000,
            input_cost_per_1k=0.00015, output_cost_per_1k=0.0006,
            reasoning_score=6, tool_reliability=0.85, speed_score=9
        ),
        "gpt-4o": ModelCapability(
            name="gpt-4o", context_window=128000,
            input_cost_per_1k=0.005, output_cost_per_1k=0.015,
            reasoning_score=9, tool_reliability=0.95, speed_score=8
        ),
        "claude-3-5-sonnet": ModelCapability(
            name="claude-3-5-sonnet-20241022", context_window=200000,
            input_cost_per_1k=0.003, output_cost_per_1k=0.015,
            reasoning_score=9, tool_reliability=0.93, speed_score=7
        ),
    }
    
    def route(
        self,
        complexity: TaskComplexity,
        max_budget_per_request: float = 0.01,
        min_quality_score: float = 0.8,
    ) -> str:
        candidates = []
        
        for name, cap in self.MODELS.items():
            # 估算单次请求成本（假设4K input, 1K output）
            estimated_cost = (
                4 * cap.input_cost_per_1k + 
                1 * cap.output_cost_per_1k
            ) / 1000
            
            # 质量评分
            quality = (cap.reasoning_score / 10 * 0.6 + 
                      cap.tool_reliability * 0.4)
            
            # 筛选条件
            if estimated_cost > max_budget_per_request:
                continue
            if quality < min_quality_score:
                continue
            
            # 计算性价比
            value = quality / max(estimated_cost, 0.0001)
            candidates.append((name, value, estimated_cost))
        
        if not candidates:
            # 放宽预算限制
            return min(self.MODELS.keys(), key=lambda k: self.MODELS[k].input_cost_per_1k)
        
        # 按性价比排序
        candidates.sort(key=lambda x: x[1], reverse=True)
        return candidates[0][0]

# 使用
router = ModelRouter()
print(router.route(TaskComplexity.SIMPLE, max_budget_per_request=0.001))
print(router.route(TaskComplexity.COMPLEX, max_budget_per_request=0.05))
```

## 第2章 Transformer架构核心原理

### 2.1 注意力机制：Self-Attention与Multi-Head Attention

#### 2.1.1 注意力机制的直觉理解

注意力机制的灵感来源于人类视觉注意力：当我们看一张图片时，不会均匀处理所有像素，而是聚焦于关键区域。在语言处理中，注意力让模型在处理每个词时，能够"关注"句子中其他相关的词。

```
句子："猫坐在垫子上，因为它很温暖"

处理"它"时，注意力权重：
- "它" → "它"     : 0.15
- "它" → "垫子"   : 0.55  ← 最高！
- "它" → "温暖"   : 0.20
- "它" → "猫"     : 0.05
- "它" → "坐"     : 0.03
- "它" → "在"     : 0.01
- "它" → "上"     : 0.01
```

#### 2.1.2 Scaled Dot-Product Attention的数学

```python
import torch
import torch.nn as nn
import math

class ScaledDotProductAttention(nn.Module):
    """
    缩放点积注意力：Transformer的核心计算单元。
    
    数学公式：
    Attention(Q, K, V) = softmax(QK^T / √d_k) V
    
    其中：
    - Q (Query): 当前位置"查询"其他位置的信息
    - K (Key): 每个位置的"索引"，用于被查询
    - V (Value): 每个位置的实际"内容"
    - d_k: Key向量的维度，缩放因子防止softmax饱和
    """
    
    def __init__(self, d_model: int = 512):
        super().__init__()
        self.d_model = d_model
        self.d_k = d_model  # 简化为相同维度
    
    def forward(self, Q, K, V, mask=None):
        """
        Args:
            Q, K, V: [batch_size, seq_len, d_model]
            mask: [batch_size, seq_len, seq_len] 或 None
        
        Returns:
            output: [batch_size, seq_len, d_model]
            attention_weights: [batch_size, seq_len, seq_len]
        """
        # 1. 计算注意力分数: Q @ K^T
        # Q: [B, L, D], K^T: [B, D, L]
        # scores: [B, L, L]
        scores = torch.matmul(Q, K.transpose(-2, -1))
        
        # 2. 缩放：除以 √d_k
        # 原因：当d_k较大时，点积值会很大，导致softmax梯度消失
        scores = scores / math.sqrt(self.d_k)
        
        # 3. 应用掩码（因果掩码或填充掩码）
        if mask is not None:
            # mask为0的位置填充负无穷，softmax后变为0
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        # 4. Softmax：转换为概率分布
        # 每行之和为1
        attention_weights = torch.softmax(scores, dim=-1)
        
        # 5. 加权求和：attention_weights @ V
        # attention_weights: [B, L, L], V: [B, L, D]
        # output: [B, L, D]
        output = torch.matmul(attention_weights, V)
        
        return output, attention_weights

# 演示
B, L, D = 2, 4, 8  # batch=2, seq_len=4, dim=8
Q = torch.randn(B, L, D)
K = torch.randn(B, L, D)
V = torch.randn(B, L, D)

# 因果掩码：下三角矩阵（只能看到当前位置及之前）
causal_mask = torch.tril(torch.ones(L, L)).unsqueeze(0)  # [1, L, L]

attn = ScaledDotProductAttention(D)
output, weights = attn(Q, K, V, causal_mask)

print(f"Q shape: {Q.shape}")
print(f"Attention weights shape: {weights.shape}")
print(f"Output shape: {output.shape}")
print(f"\n因果掩码后的注意力权重（第一样本）：")
print(weights[0].round(3))
# 注意：每行只在前i+1列有非零值（因果性）
```

#### 2.1.3 Multi-Head Attention

```python
class MultiHeadAttention(nn.Module):
    """
    多头注意力：将注意力计算拆分为多个并行的"头"。
    
    核心思想：
    - 不同头学习不同的注意力模式
    - 例如：一个头关注语法关系，另一个关注语义关系
    - 最后将各头的输出拼接并线性变换
    
    数学：
    MultiHead(Q,K,V) = Concat(head_1, ..., head_h) W^O
    where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
    """
    
    def __init__(self, d_model: int = 512, num_heads: int = 8):
        super().__init__()
        assert d_model % num_heads == 0, "d_model必须能被num_heads整除"
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 每个头的维度
        
        # 线性投影层
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        
        self.attention = ScaledDotProductAttention(self.d_k)
    
    def forward(self, Q, K, V, mask=None):
        B = Q.size(0)
        
        # 1. 线性投影并分头
        # [B, L, D] -> [B, L, H, d_k] -> [B, H, L, d_k]
        Q_proj = self.W_q(Q).view(B, -1, self.num_heads, self.d_k).transpose(1, 2)
        K_proj = self.W_k(K).view(B, -1, self.num_heads, self.d_k).transpose(1, 2)
        V_proj = self.W_v(V).view(B, -1, self.num_heads, self.d_k).transpose(1, 2)
        
        # 2. 对每个头分别计算注意力
        # 如果有mask，需要扩展维度
        if mask is not None:
            mask = mask.unsqueeze(1)  # [B, 1, L, L]
        
        attn_output, weights = self.attention(Q_proj, K_proj, V_proj, mask)
        
        # 3. 拼接多头输出
        # [B, H, L, d_k] -> [B, L, H, d_k] -> [B, L, D]
        attn_output = attn_output.transpose(1, 2).contiguous().view(B, -1, self.d_model)
        
        # 4. 最终线性变换
        output = self.W_o(attn_output)
        
        return output

# 演示
mha = MultiHeadAttention(d_model=512, num_heads=8)
x = torch.randn(2, 10, 512)  # [batch=2, seq=10, dim=512]
out = mha(x, x, x)  # 自注意力：Q=K=V
print(f"MHA输出: {out.shape}")  # [2, 10, 512]
```

#### 2.1.4 位置编码

Transformer本身没有序列顺序的概念（与RNN不同），需要通过位置编码注入位置信息。

```python
class SinusoidalPositionalEncoding(nn.Module):
    """
    原始Transformer使用的正弦位置编码。
    
    公式：
    PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
    PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
    
    优点：
    - 可以处理任意长度的序列
    - 相对位置可以线性表达：PE(pos+k)可以表示为PE(pos)的线性函数
    """
    
    def __init__(self, d_model: int, max_len: int = 5000):
        super().__init__()
        
        # 预计算位置编码矩阵
 pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * 
            (-math.log(10000.0) / d_model)
        )
        
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        
        self.register_buffer('pe', pe.unsqueeze(0))
    
    def forward(self, x):
        """
        x: [batch_size, seq_len, d_model]
        """
        return x + self.pe[:, :x.size(1), :]

class RotaryPositionalEmbedding(nn.Module):
    """
    RoPE（旋转位置编码）：现代LLM（Llama、Mistral等）使用。
    
    核心思想：通过旋转矩阵编码相对位置，
    使得内积自然体现相对位置信息。
    """
    
    def __init__(self, dim: int, max_seq_len: int = 2048):
        super().__init__()
        inv_freq = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer("inv_freq", inv_freq)
    
    def forward(self, seq_len: int, device):
        t = torch.arange(seq_len, device=device, dtype=self.inv_freq.dtype)
        freqs = torch.einsum("i,j->ij", t, self.inv_freq)
        emb = torch.cat((freqs, freqs), dim=-1)
        return emb.cos(), emb.sin()
```

### 2.2 为什么GPT系列适合Agent任务

```python
"""
三种Transformer架构对比：

1. 编码器-only（Encoder-only）
   代表：BERT, RoBERTa, DeBERTa
   结构：双向注意力（每个token可以看到左右所有token）
   优势：理解任务（分类、NER、问答）
   劣势：不能生成文本
   Agent适用性：❌ 无法自主产生输出

2. 编码器-解码器（Encoder-Decoder）
   代表：T5, BART, UL2
   结构：编码器双向 + 解码器单向
   优势：翻译、摘要、格式转换
   劣势：对Agent的循环推理支持不够好
   Agent适用性：⚠️ 可以做，但不是最优

3. 解码器-only（Decoder-only）
   代表：GPT, Llama, Claude, Mistral
   结构：因果注意力（只能看到左边token）
   优势：
     - 自回归生成：天然适合逐步推理
     - 统一接口：输入输出都是文本序列
     - In-Context Learning：通过prompt示例学习
     - 工具调用格式：生成JSON或特定格式文本
   Agent适用性：✅ 最佳选择

为什么Decoder-only更适合Agent？

1. 自回归 = 逐步思考
   Agent的推理过程：思考 → 行动 → 观察 → 再思考...
   这与自回归生成（一个一个token生成）天然匹配。

2. 统一的token空间
   所有信息（指令、历史、工具结果）都表示为token序列，
   模型用同一种机制处理所有内容。

3. In-Context Learning
   通过prompt中的示例，模型可以"学会"使用新工具，
   无需微调。这是Agent快速扩展能力的基础。

4. 涌现的推理能力
   当模型规模达到一定程度（约100B参数），
   链式思考（Chain-of-Thought）能力涌现。
"""
```

```python
def parse_chat_response(response) -> dict:
    """
    解析OpenAI Chat Completion响应的所有可能情况。
    """
    if not response.choices:
        return {"type": "empty", "reason": "No choices returned"}
    
    choice = response.choices[0]
    message = choice.message
    finish_reason = choice.finish_reason
    
    result = {
        "finish_reason": finish_reason,
        "model": response.model,
        "usage": {
            "prompt_tokens": response.usage.prompt_tokens,
            "completion_tokens": response.usage.completion_tokens,
            "total_tokens": response.usage.total_tokens,
        }
    }
    
    # 情况1：正常文本回复
    if message.content and not message.tool_calls:
        result["type"] = "content"
        result["content"] = message.content
        return result
    
    # 情况2：工具调用
    if message.tool_calls:
        result["type"] = "tool_calls"
        result["tool_calls"] = [
            {
                "id": tc.id,
                "name": tc.function.name,
                "arguments": json.loads(tc.function.arguments)
            }
            for tc in message.tool_calls
        ]
        return result
    
    # 情况3：内容过滤
    if finish_reason == "content_filter":
        result["type"] = "blocked"
        result["reason"] = "Content filtered by safety system"
        return result
    
    # 情况4：长度截断
    if finish_reason == "length":
        result["type"] = "truncated"
        result["content"] = message.content or ""
        result["reason"] = "Max tokens reached"
        return result
    
    # 情况5：空响应（模型拒绝）
    if not message.content and not message.tool_calls:
        result["type"] = "refusal"
        result["reason"] = "Model refused to respond"
        return result
    
    return result
```

### 3.3 流式响应处理

```python
from typing import AsyncIterator

async def stream_chat_completion(
    client: openai.AsyncOpenAI,
    messages: list[dict],
    model: str = "gpt-4o",
    tools: Optional[list] = None,
) -> AsyncIterator[dict]:
    """
    流式处理Chat Completion，实时解析事件类型。
    
    产生的事件类型：
    - {"type": "token", "content": "..."}           普通文本token
    - {"type": "tool_call_start", "id": "..."}       工具调用开始
    - {"type": "tool_call_delta", "id": "...", ...}  工具调用增量
    - {"type": "tool_call_end", "id": "..."}         工具调用结束
    - {"type": "done"}                               流结束
    """
    stream = await client.chat.completions.create(
        model=model,
        messages=messages,
        tools=tools,
        stream=True,
        stream_options={"include_usage": True} if tools else None,
    )
    
    current_tool_calls = {}
    
    async for chunk in stream:
        delta = chunk.choices[0].delta if chunk.choices else None
        
        if not delta:
            if hasattr(chunk, 'usage') and chunk.usage:
                yield {"type": "usage", "usage": chunk.usage}
            continue
        
        # 处理文本内容
        if delta.content:
            yield {"type": "token", "content": delta.content}
        
        # 处理工具调用（增量更新）
        if delta.tool_calls:
            for tc_delta in delta.tool_calls:
                idx = tc_delta.index
                
                if idx not in current_tool_calls:
                    current_tool_calls[idx] = {
                        "id": "", "name": "", "arguments": ""
                    }
                    yield {
                        "type": "tool_call_start",
                        "index": idx
                    }
                
                tc = current_tool_calls[idx]
                
                if tc_delta.id:
                    tc["id"] += tc_delta.id
                if tc_delta.function and tc_delta.function.name:
                    tc["name"] += tc_delta.function.name
                    yield {
                        "type": "tool_call_name",
                        "index": idx,
                        "name": tc["name"]
                    }
                if tc_delta.function and tc_delta.function.arguments:
                    tc["arguments"] += tc_delta.function.arguments
                    yield {
                        "type": "tool_call_args_delta",
                        "index": idx,
                        "arguments": tc["arguments"]
                    }
        
        # 检查finish_reason
        if chunk.choices[0].finish_reason:
            # 完成所有工具调用
            for idx, tc in current_tool_calls.items():
                try:
                    args = json.loads(tc["arguments"])
                except:
                    args = {}
                yield {
                    "type": "tool_call_end",
                    "index": idx,
                    "tool_call": {
                        "id": tc["id"],
                        "name": tc["name"],
                        "arguments": args
                    }
                }
            
            yield {"type": "done", "finish_reason": chunk.choices[0].finish_reason}
```

### 3.4 多模型路由与Fallback

```python
import asyncio
from dataclasses import dataclass
from typing import AsyncIterator, Optional
import openai

@dataclass
class ProviderConfig:
    name: str
    client: openai.AsyncOpenAI
    model: str
    priority: int          # 优先级，数字越小越优先
    timeout: float = 30.0
    weight: float = 1.0    # 负载均衡权重

class ResilientLLMRouter:
    """
    弹性LLM路由器：支持多Provider fallback和流式响应。
    
    特性：
    1. 优先级路由：按优先级尝试不同Provider
    2. 健康检查：自动检测故障Provider并暂时屏蔽
    3. 流式fallback：流式响应中断时可切换到其他Provider
    4. 成本意识：可配置成本优先级
    """
    
    def __init__(self, providers: list[ProviderConfig]):
        self.providers = sorted(providers, key=lambda p: p.priority)
        self.health_status: dict[str, bool] = {p.name: True for p in providers}
        self.failure_counts: dict[str, int] = {p.name: 0 for p in providers}
        self.circuit_threshold = 3  # 连续失败3次后熔断
    
    def _mark_failure(self, provider_name: str):
        self.failure_counts[provider_name] += 1
        if self.failure_counts[provider_name] >= self.circuit_threshold:
            self.health_status[provider_name] = False
            print(f"⚠️ Provider {provider_name} 已熔断")
    
    def _mark_success(self, provider_name: str):
        self.failure_counts[provider_name] = 0
        self.health_status[provider_name] = True
    
    def _get_healthy_providers(self) -> list[ProviderConfig]:
        return [p for p in self.providers if self.health_status[p.name]]
    
    async def chat_with_fallback(
        self,
        messages: list[dict],
        temperature: float = 0.7,
        max_tokens: int = 1000,
        tools: Optional[list] = None,
    ) -> dict:
        """非流式调用，自动fallback"""
        last_error = None
        
        for provider in self._get_healthy_providers():
            try:
                response = await asyncio.wait_for(
                    provider.client.chat.completions.create(
                        model=provider.model,
                        messages=messages,
                        temperature=temperature,
                        max_tokens=max_tokens,
                        tools=tools,
                    ),
                    timeout=provider.timeout
                )
                self._mark_success(provider.name)
                return {
                    "success": True,
                    "provider": provider.name,
                    "model": provider.model,
                    "response": response
                }
            except Exception as e:
                self._mark_failure(provider.name)
                last_error = f"{provider.name}: {str(e)}"
                continue
        
        return {
            "success": False,
            "error": f"All providers failed. Last: {last_error}"
        }
    
    async def stream_with_fallback(
        self,
        messages: list[dict],
        temperature: float = 0.7,
        max_tokens: int = 1000,
    ) -> AsyncIterator[dict]:
        """流式调用，如果当前Provider中断，尝试下一个"""
        for provider in self._get_healthy_providers():
            try:
                stream = await provider.client.chat.completions.create(
                    model=provider.model,
                    messages=messages,
                    temperature=temperature,
                    max_tokens=max_tokens,
                    stream=True,
                )
                
                async for chunk in stream:
                    yield {
                        "type": "token",
                        "provider": provider.name,
                        "chunk": chunk
                    }
                
                self._mark_success(provider.name)
                yield {"type": "done", "provider": provider.name}
                return
                
            except Exception as e:
                self._mark_failure(provider.name)
                yield {
                    "type": "provider_failed",
                    "provider": provider.name,
                    "error": str(e)
                }
        
        yield {"type": "all_failed"}
```

### 4.1 Tokenization原理

```python
"""
Tokenization：将文本拆分为模型可处理的整数序列。

三种主流算法：

1. BPE (Byte-Pair Encoding)
   代表：GPT系列、Llama
   过程：
   - 从字符级别词表开始
   - 统计最常见的字符对，合并为新token
   - 重复直到词表达到目标大小
   
   特点：
   - 英文：约4字符/token
   - 中文：约1-1.5字符/token（中文一个字通常就是一个token）
   - 数字：每个数字通常单独一个token

2. WordPiece
   代表：BERT
   类似BPE，但使用概率而不是频率决定合并

3. SentencePiece
   代表：T5、Llama
   特点：
   - 直接在原始文本（包括空格）上训练
   - 不需要预分词
   - 对多语言更友好
"""

import tiktoken

    print(f"字符数量: {len(text)}")
    print(f"字符/Token比: {len(text) / max(len(tokens), 1):.2f}")
    print(f"\nToken分解:")
    
    for i, token_id in enumerate(tokens):
```

### 4.2 Token计数与预算管理

```python
import tiktoken
from dataclasses import dataclass
from typing import Optional

@dataclass
class TokenBudget:
    """Token预算管理器"""
    total_budget: int
    system_reserve: int = 500      # 保留给系统提示
    response_reserve: int = 1000   # 保留给回复
    
    @property
    def available_for_context(self) -> int:
        return self.total_budget - self.system_reserve - self.response_reserve

class TokenCounter:
    """
    精确的Token计数器。
    
    为什么需要这个？
    - 不同模型的tokenizer不同
    - 消息格式（role、name等）也会占用token
    - 需要精确控制以避免超出上下文窗口
    """
    
    def __init__(self, model: str = "gpt-4"):
        self.encoder = tiktoken.encoding_for_model(model)
        self.model = model
    
    def count_message(self, message: dict) -> int:
        """计算单条消息的token数（包含格式开销）"""
        # OpenAI的消息格式token开销：
        # 每条消息有固定的格式开销
        tokens_per_message = 3  # <|start|>{role}\n{content}<|end|>\n
        tokens = tokens_per_message
        
        for key, value in message.items():
            if value:
                tokens += len(self.encoder.encode(str(value)))
                if key == "name":
                    tokens += 1  # name有额外开销
        
        return tokens
    
    def count_messages(self, messages: list[dict]) -> int:
        """计算消息列表的总token数"""
        total = 3  # priming token（所有对话的固定前缀）
        for msg in messages:
            total += self.count_message(msg)
        return total
    
    def count_text(self, text: str) -> int:
        return len(self.encoder.encode(text))
    
    def trim_messages_to_fit(
        self,
        messages: list[dict],
        max_tokens: int,
        keep_system: bool = True,
    ) -> list[dict]:
        """
        裁剪消息列表以适应token限制。
        
        策略：
        1. 保留系统消息（如果keep_system=True）
        2. 从最早的消息开始删除
        3. 保留最近的几轮对话
        """
        result = messages.copy()
        
        while self.count_messages(result) > max_tokens and len(result) > 1:
            if keep_system and result[0].get("role") == "system":
                # 删除第二条消息（保留系统消息）
                if len(result) > 1:
                    del result[1]
                else:
                    break
            else:
                del result[0]
        
        return result

# 使用

    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "Hello" * 500},
    {"role": "assistant", "content": "Hi!" * 500},
]

print(f"总token: {counter.count_messages(messages)}")
```

### 4.3 上下文窗口与长文本策略

```python
"""
上下文窗口的物理限制：

Transformer的注意力计算复杂度是 O(n²)，其中n是序列长度。
这意味着：
- 计算量随长度平方增长
- KV Cache（键值缓存）随长度线性增长
- 内存消耗是瓶颈

各模型的上下文窗口：
- GPT-4o: 128K tokens ≈ 200页英文文档
- Claude 3.5: 200K tokens ≈ 300页英文文档
- Gemini 1.5 Pro: 1M+ tokens ≈ 1500页英文文档
- Llama 3.1: 128K tokens

长文本处理策略：
"""

from typing import Callable
import hashlib

class ContextManager:
    """
    Agent上下文管理器：处理长对话历史。
    
    策略组合：
    1. 滑动窗口：保留最近N轮
    2. 摘要压缩：将旧对话压缩为摘要
    3. 向量检索：按需检索相关历史
    """
    
    def __init__(self, max_tokens: int = 8000, counter: Optional[TokenCounter] = None):
        self.max_tokens = max_tokens
        self.counter = counter or TokenCounter()
        self.raw_history: list[dict] = []
        self.summaries: list[str] = []
    
    def add_message(self, message: dict):
        """添加消息到历史"""
        self.raw_history.append(message)
    
    def get_context_for_llm(self) -> list[dict]:
        """获取适合传入LLM的上下文"""
        # 策略1：如果总token在限制内，返回全部
        total = self.counter.count_messages(self.raw_history)
        if total <= self.max_tokens:
            return self.raw_history
        
        # 策略2：滑动窗口 + 摘要
        return self._build_window_with_summary()
    
    def _build_window_with_summary(self) -> list[dict]:
        """构建摘要 + 最近对话的上下文"""
        system_msgs = [m for m in self.raw_history if m.get("role") == "system"]
        other_msgs = [m for m in self.raw_history if m.get("role") != "system"]
        
        # 保留最近6轮对话（12条消息）
        recent = other_msgs[-12:]
        older = other_msgs[:-12]
        
        # 将旧对话压缩为摘要
        if older and not self.summaries:
            summary = self._summarize_messages(older)
            self.summaries.append(summary)
        
        context = system_msgs.copy()
        if self.summaries:
            context.append({
                "role": "system",
                "content": f"历史对话摘要：{self.summaries[-1]}"
            })
        context.extend(recent)
        
        # 如果还超，进一步裁剪
        while self.counter.count_messages(context) > self.max_tokens and len(context) > len(system_msgs) + 1:
            # 删除最早的非系统消息
            for i, msg in enumerate(context):
                if msg.get("role") != "system":
                    del context[i]
                    break
        
        return context
    
    def _summarize_messages(self, messages: list[dict]) -> str:
        """将消息列表总结为一句话摘要"""
        # 简化实现：实际应调用LLM进行摘要
        user_msgs = [m["content"][:50] for m in messages if m.get("role") == "user"]
        return f"此前用户询问了：{'；'.join(user_msgs[:3])}等"
    
    def clear(self):
        self.raw_history = []
        self.summaries = []
```

### 5.1 上下文窗口的物理限制

```python
"""
KV Cache：长上下文的内存杀手

Transformer解码时，为了避免重复计算，会缓存之前token的Key和Value向量。
这就是KV Cache。

内存计算：
- 每层有 num_heads 个注意力头
- 每个头有 d_k 维度
- KV Cache大小 = 2 * num_layers * num_heads * d_k * seq_len * batch_size * sizeof(float)

以Llama-3-70B为例：
- layers=80, heads=64, d_k=128, batch=1
- 每token的KV Cache = 2 * 80 * 64 * 128 * 4 bytes = 5.24 MB
- 128K上下文的KV Cache = 5.24 MB * 128K = 655 GB！

解决方案：
1. KV Cache量化（INT8/INT4）：减少50-75%内存
2. 滑动窗口注意力：只缓存最近W个token
3. 分组查询注意力（GQA）：多个头共享KV
4. 分页KV Cache（vLLM的PagedAttention）
"""
```

```python
from typing import List

class ContextCompressor:
    """
    上下文压缩器：多种策略减少token使用量。
    """
    
    @staticmethod
    def remove_cot_thinking(text: str, markers: tuple = ("<think>", "</think>")) -> str:
        """移除Chain-of-Thought的思考过程，只保留结论"""
        import re
        pattern = f"{re.escape(markers[0])}.*?{re.escape(markers[1])}"
        return re.sub(pattern, "", text, flags=re.DOTALL)
    
    @staticmethod
    def truncate_tool_results(
        messages: List[dict],
        max_tool_result_length: int = 500
    ) -> List[dict]:
        """截断过长的工具返回结果"""
        result = []
        for msg in messages:
            if msg.get("role") == "tool" and len(msg.get("content", "")) > max_tool_result_length:
                content = msg["content"]
                truncated = content[:max_tool_result_length] + f"\n... [截断，原长度{len(content)}字符]"
                result.append({**msg, "content": truncated})
            else:
                result.append(msg)
        return result
    
    @staticmethod
    def deduplicate_messages(messages: List[dict]) -> List[dict]:
        """去重连续重复的相同消息"""
        if not messages:
            return messages
        
        result = [messages[0]]
        for msg in messages[1:]:
            if not (msg.get("role") == result[-1].get("role") and 
                    msg.get("content") == result[-1].get("content")):
                result.append(msg)
        return result

# 测试
```

```python
import asyncio
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import AsyncIterator, Optional, List, Dict, Any
from enum import Enum
import time

class ModelProvider(Enum):
    OPENAI = "openai"
    ANTHROPIC = "anthropic"
    AZURE = "azure"
    OLLAMA = "ollama"
    VLLM = "vllm"

@dataclass
class UnifiedResponse:
    content: str
    model: str
    provider: ModelProvider
    input_tokens: int
    output_tokens: int
    latency_ms: float
    tool_calls: Optional[List[Dict]] = None
    finish_reason: Optional[str] = None

class BaseLLMProvider(ABC):
    """统一的LLM提供商接口"""
    
    def __init__(self, model: str, timeout: float = 30.0):
        self.model = model
        self.timeout = timeout
    
    @abstractmethod
    async def chat(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 1000,
        tools: Optional[List[Dict]] = None,
    ) -> UnifiedResponse:
        pass
    
    @abstractmethod
    async def stream_chat(
        self,
        messages: List[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 1000,
    ) -> AsyncIterator[str]:
        pass

class OpenAIProvider(BaseLLMProvider):
    def __init__(self, api_key: str, model: str = "gpt-4o"):
        super().__init__(model)
        import openai
        self.client = openai.AsyncOpenAI(api_key=api_key)
        self.provider = ModelProvider.OPENAI
    
    async def chat(self, messages, temperature=0.7, max_tokens=1000, tools=None):
        start = time.time()
        
        response = await asyncio.wait_for(
            self.client.chat.completions.create(
                model=self.model,
                messages=messages,
                temperature=temperature,
                max_tokens=max_tokens,
                tools=tools,
            ),
            timeout=self.timeout
        )
        
        latency = (time.time() - start) * 1000
        choice = response.choices[0]
        
        return UnifiedResponse(
            content=choice.message.content or "",
            model=self.model,
            provider=self.provider,
            input_tokens=response.usage.prompt_tokens,
            output_tokens=response.usage.completion_tokens,
            latency_ms=latency,
            tool_calls=[
                {
                    "id": tc.id,
                    "name": tc.function.name,
                    "arguments": tc.function.arguments
                }
                for tc in (choice.message.tool_calls or [])
            ] or None,
            finish_reason=choice.finish_reason,
        )
    
    async def stream_chat(self, messages, temperature=0.7, max_tokens=1000):
        stream = await self.client.chat.completions.create(
            model=self.model,
            messages=messages,
            temperature=temperature,
            max_tokens=max_tokens,
            stream=True,
        )
        async for chunk in stream:
            if chunk.choices[0].delta.content:
                yield chunk.choices[0].delta.content

class OllamaProvider(BaseLLMProvider):
    def __init__(self, base_url: str = "http://localhost:11434", model: str = "llama3.1"):
        super().__init__(model)
        self.base_url = base_url
        self.provider = ModelProvider.OLLAMA
    
    async def chat(self, messages, temperature=0.7, max_tokens=1000, tools=None):
        import aiohttp
        start = time.time()
        
        async with aiohttp.ClientSession() as session:
            async with session.post(
                f"{self.base_url}/api/chat",
                json={
                    "model": self.model,
                    "messages": messages,
                    "stream": False,
                    "options": {
                        "temperature": temperature,
                        "num_predict": max_tokens,
                    }
                }
            ) as resp:
                data = await resp.json()
        
        latency = (time.time() - start) * 1000
        
        return UnifiedResponse(
            content=data["message"]["content"],
            model=self.model,
            provider=self.provider,
            input_tokens=data.get("prompt_eval_count", 0),
            output_tokens=data.get("eval_count", 0),
            latency_ms=latency,
        )
    
    async def stream_chat(self, messages, temperature=0.7, max_tokens=1000):
        import aiohttp
        import json
        
        async with aiohttp.ClientSession() as session:
            async with session.post(
                f"{self.base_url}/api/chat",
                json={
                    "model": self.model,
                    "messages": messages,
                    "stream": True,
                    "options": {"temperature": temperature}
                }
            ) as resp:
                async for line in resp.content:
                    if line:
                        try:
                            data = json.loads(line)
                            if "message" in data and "content" in data["message"]:
                                yield data["message"]["content"]
                        except:
                            pass

class UnifiedLLMClient:
    """
    统一的LLM客户端：屏蔽不同提供商的差异。
    
    支持：
    - 多Provider配置
    - 自动fallback
    - 统一响应格式
    - 流式/非流式
    - Token使用统计
    """
    
    def __init__(self):
        self.providers: Dict[str, BaseLLMProvider] = {}
        self.fallback_order: List[str] = []
    
    def register_provider(self, name: str, provider: BaseLLMProvider, priority: int = 0):
        self.providers[name] = provider
        self.fallback_order = sorted(
            self.providers.keys(),
            key=lambda k: priority if k == name else 0
        )
    
    async def chat(
        self,
        messages: List[Dict],
        provider: Optional[str] = None,
        **kwargs
    ) -> UnifiedResponse:
        """发送聊天请求，支持自动fallback"""
        providers_to_try = [provider] if provider else self.fallback_order
        last_error = None
        
        for name in providers_to_try:
            if name not in self.providers:
                continue
            
            p = self.providers[name]
            try:
                return await p.chat(messages, **kwargs)
            except Exception as e:
                last_error = f"{name}: {str(e)}"
                continue
        
        raise RuntimeError(f"All providers failed. Last: {last_error}")
    
    async def stream_chat(self, messages: List[Dict], provider: Optional[str] = None, **kwargs):
        p = self.providers.get(provider or self.fallback_order[0])
        if not p:
            raise ValueError("No provider available")
        async for token in p.stream_chat(messages, **kwargs):
            yield token
```
