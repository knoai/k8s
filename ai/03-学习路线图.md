## 阶段二：Kubernetes 核心（第 3-5 个月）

### 目标
掌握 K8s 核心概念与运维能力，通过 CKA 认证。

### 第 9-10 周：核心对象深度理解

| 对象 | 学习深度 | 关键概念 | 实践 |
|------|----------|----------|------|
| **Pod** | ⭐⭐⭐⭐⭐ | 生命周期、探针、Init 容器、Sidecar | 部署一个多容器 Pod |
| **Deployment** | ⭐⭐⭐⭐⭐ | 滚动更新、回滚、扩缩容、暂停/恢复 | 实现零停机发布 |
| **StatefulSet** | ⭐⭐⭐⭐ | 稳定网络标识、持久化存储、有序部署 | 部署有状态应用 |
| **DaemonSet** | ⭐⭐⭐⭐ | 节点级部署、容忍、更新策略 | 部署日志收集 Agent |
| **Job/CronJob** | ⭐⭐⭐⭐⭐ | 并行度、完成数、TTL、定时规则 | 批量数据处理任务 |
| **Service** | ⭐⭐⭐⭐⭐ | ClusterIP/NodePort/LoadBalancer/Headless | 服务发现实验 |
| **Ingress** | ⭐⭐⭐⭐ | 路由规则、TLS、Annotations | 配置 HTTPS 入口 |

### 第 11-12 周：调度与资源

| 主题 | 内容 | 实践 |
|------|------|------|
| Label/Selector | 标签设计、选择器语法 | 按环境/版本/团队标记资源 |
| Node Affinity | 硬/软亲和性、权重 | 将 GPU 任务调度到 GPU 节点 |
| Pod Affinity/Anti-Affinity | 同区域/反同一节点 | 数据库主从反亲和部署 |
| Taints/Tolerations | 污点类型、容忍配置 | GPU 节点专用化 |
| ResourceQuota | 命名空间资源限制 | 为团队分配资源配额 |
| LimitRange | 默认资源限制 | 防止资源申请失控 |

### 第 13-14 周：存储与网络

| 主题 | 内容 | 实践 |
|------|------|------|
| PV/PVC/StorageClass | 静态/动态供给、绑定模式 | 为应用配置持久化存储 |
| ConfigMap/Secret | 配置注入、加密、环境变量 | 管理应用配置 |
| CNI 基础 | Flannel/Calico/Cilium 原理 | 查看集群 CNI 配置 |
| NetworkPolicy | 入站/出站规则、命名空间隔离 | 实现微服务网络隔离 |
| DNS | CoreDNS 配置、自定义域名 | 配置服务发现 |

### 第 15-16 周：高级主题

| 主题 | 内容 | 实践 |
|------|------|------|
| RBAC | Role/ClusterRole/Binding | 配置最小权限 |
| Admission Controller | Validating/Mutating Webhook | 理解 OPA/Kyverno 原理 |
| CRD | 自定义资源定义 | 创建一个简单 CRD |
| Operator | 控制器模式、Reconcile 循环 | 了解 Operator SDK |
| Helm | Chart 结构、模板语法、Values | 编写自定义 Chart |

### 第 17-20 周：CKA 认证备考

| 周次 | 内容 |
|------|------|
| 17 | 官方教程通读 + 实验环境搭建 |
| 18 | Killer.sh 模拟题（第一次，摸底） |
| 19 | 薄弱项专项训练 + 第二次模拟 |
| 20 | 第三次模拟 + 预约考试 |

**CKA 备考资源优先级**：
1. KodeKloud CKA 课程（系统学习）
2. Killer.sh 模拟考试（最接近真题）
3. Kubernetes 官方文档（考试允许查阅）
4. 官方练习环境（Play with Kubernetes）

**检验标准**：
- [ ] 通过 CKA 认证
- [ ] 能独立部署一个多服务 Web 应用（含 Ingress、存储、配置）
- [ ] 能排查常见 K8s 故障（Pod 起不来、网络不通、存储挂载失败）
